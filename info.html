Usage Questions

How often does the data of a given user update?

Whenever you want. Every page reload will update the data, you can even click on the user profile and press sync badges to update without reloading.
There will never be a feature to auto-refresh every X minutes. One reload is cheaper than this.

What does it mean if some of the badges are cyan/blue?

If a badge has been claimed within the past 4 weeks, it will be a shade of cyan instead of green. This allows easy access to see which badges you have most recently completed.
Note: This feature is based off Refined Github File Age feature

Why do some categories have rainbow text?

After a category is 100% completed (of the badges displayed), the text will become rainbow to represent that. This is also a feature in-game.

How do i load another user?

You should have already seen the Load User navigation on your way here... Just click that.
Alternatively, desktop users can click the username under Currently Viewing: as a "quick-load" per-say.

Why can't i load a user by display name?

Because roblox doesn't have an API to do so, we would be shooting in the dark and might not even get the desired user. Also, unlike usernames. Display names are not tied to the user, someone else can claim it hence your user might just change randomly...
The best we can do, is initial username load and then display name option as long as the user is downloaded... But that is more work and still contains problems outside our control.

Why do some users take longer to load than others?

EToH Contains over 300 towers, and 2 games worth of badges. In total thats like 600+ badges we have to load. For the client, i've tried to make it as quick as possible with the ability of streaming badges. But on cloudflare, i still have to make 6+ network requests.
Once a user has been stored to the device, we can reduce the amount of badges we request hence loading will be quicker.

What is ?user={name} and why is it there?

This is automatically added by the site, expect for when loading the default user. This short-code is a way to tell the site what user you wish to load and can be easily shared.

What is a Default User?

The ability to set a default user can be done by clicking on the user profile, then Set default user. This means anytime you come to this site without the user URL parameter. ?user={name}, that user will be loaded instead.
Do note, only one user can be the default at any given time.

Does the site have mobile support?

Yes! It won't be the best experience but it'll be a better experience than just trying to use the desktop site on a smaller screen. Please file issues on github to help out though.

How do i access the browser search?

Just use the same keybind again when focusing the site search. Or use an alternative keybind / method your browser has.

Technically Questions

How badges are updated

Every day, github runs some code to update the badges. If a badge has a wiki counter-part, we use data scrapped off the wiki. If it does not, then it will check a premade file.
If a new badge doesn't have a wiki counterpart, or something goes wrong. I get an email of the issue and will attempt to fix it within the day.

Why use external sources for badges?

The wiki is updated by the community and is a "source of truth" which everyone relays on. If that is wrong, then everything is wrong. We could get them from in-game, but thats a more complex setup.
I am scraping without permission, i know. I won't stop though as this is just too convenient. I will however only do it once per day unless i need to upgrade the system which i'll do offline testing for.

If you scrap the wiki, why not scrap discord?

Discord has all of the completions in one channel, and a ton of data. Whilst we could scrap discord for said data, just to load a user will take a while depending on how many reruns of a tower they completed etc. I would basically need to host my own external database which might get expensive. Or just deal with the very long time of processing.

Additionally, due to various reasons not all completions have been submitted. so the data is unreliable anyway. Make one yourself if you really really want it.

Cloudflare?

This: https://roblox-proxy.dragmine149.workers.dev/. Due to not being able to send network requests directly to roblox from the browser, we need to go via a third party.
This also means, i can clean up some of the terrible response types they have and send back nicer responses. Fell free to use it, just don't spam it too much.

Other / legay? Questions

What data is collected?

No data is collected... Is what i would like to say, but the network requests are collected. Upon loading the site, you send stuff to github which idk what github does with. And upon loading a user, you send a couple to cloudflare.

Cloudflare requests are stored for 3 days as "logs" which i will use to debug things if user data, or something is not loading correctly.

Other than that no data is collected and there is not really any use to collecting data.

What data is stored locally?

We use the following storages for the following reasons:

  • localStorage: Storing preferences and stuff like default load user.
  • cacheStorage: Storing network requests to github to get badge data. This makes subsequential loads in the same day way quicker
  • indexedDB: Storing links between a badge and a roblox user, with basic roblox user data. This makes loading a user even quicker on subsequential loads.
Deleting any of the above will not break the website at worst things will take longer to load and settings will be reset. The amount of dynamic data in indexedDB depends on the browser. If you have partial data, the site will still be able to recover just fine.
